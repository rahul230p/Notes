## Q2: How would you validate market opportunity?

### Answer:

**Validation Approach:**

| Step | Method | Output |
|------|--------|--------|
| **Market Research** | Surveys, interviews, trend analysis | Market size estimate, demand signals |
| **Competitive Analysis** | App store reviews, pricing benchmarks | Competitive positioning |
| **Pilot/Test** | Limited rollout to subset of users | Real user behavior, conversion rates |
| **Merchant Interviews** | Outreach to potential supply partners | Supply-side enthusiasm, economics |
| **Financial Modeling** | Revenue projections, unit economics | Expected profitability, breakeven timeline |

**Pilot Design Example (New Geographic Market):**

```
PHASE 1: SOFT LAUNCH (Week 1-2)
├─ 1-2 neighborhoods, 500 early-access users
├─ 50-100 restaurants onboarded
└─ Measure: Download rate, order conversion, AOV

PHASE 2: EXPANDED BETA (Week 3-6)
├─ 5-10 neighborhoods, 50K users
├─ 500+ restaurants
└─ Measure: Retention D7/D30, repeat rate, NPS

PHASE 3: SCALE (Week 7+)
├─ City-wide rollout
├─ Aggressive marketing
└─ Measure: Market share, unit economics, profitability
```

## Q6: How would you detect if the expansion is failing early?

### Answer:

**Early Warning Signals:**

| Red Flag | Threshold | Action |
|----------|-----------|--------|
| **Low First-Order Conversion** | <10% of downloads | Revisit onboarding UX, value prop |
| **High D7 Churn** | >70% of new users inactive by Day 7 | Poor product-market fit, quality issues |
| **Low Repeat Rate** | <20% repeat rate after 30 days | Engagement problem, quality issue |
| **High CAC** | CAC > $10 (vs $5 target) | Marketing inefficiency, wrong channels |
| **Poor Order Economics** | Order margin <10% | Pricing wrong, costs too high |
| **Low Merchant Satisfaction** | Merchant churn >10%/month | Unfavorable economics for partners |
| **Quality Issues** | Rating <4.2 or on-time <90% | Operational problems |
| **Competitive Loss** | Our market share declining | Competitors gaining, we're losing differentiation |

**Recovery Actions:**

```
IF Metric Fails THEN Action:

IF First-Order Conversion < 10%
  THEN Analyze drop-off: messaging? UX? 
       A/B test value prop, simplify checkout

IF D7 Retention < 30%
  THEN Check: product quality? merchant selection?
       Consider vertical exit if unfixable

IF Repeat Rate < 20%
  THEN Launch engagement campaigns, improve quality
       Track impact week-over-week

IF Margins < 10%
  THEN Either: increase prices OR cut costs
       Model what's sustainable for market
```

## Q7: How would you structure the financial model?

### Answer:

**Unit Economics Framework:**

```
REVENUE PER ORDER
├─ Customer pays: $40 AOV
├─ DoorDash takes: 30% commission = $12
├─ Payment processor: 2.9% + $0.30 = $1.50
└─ Revenue to DoorDash: ~$10.50

COSTS PER ORDER
├─ Dasher cost: $5-7 (delivery fee)
├─ Merchant discount: $0.50-1.00 (incentive)
├─ CS/refund: $0.30
├─ Payment fraud loss: $0.20
└─ Total Cost: $6-8.50

GROSS MARGIN
├─ Revenue: $10.50
├─ Cost: $7.25 (average)
└─ Gross Margin: $3.25 (31%)

CUSTOMER-LEVEL LTV
├─ AOV: $40
├─ Order frequency: 8 orders/year (steady state)
├─ Years active: 5 years
├─ LTV: $40 × 8 × 5 × 0.31 = $496

CAC PAYBACK
├─ CAC: $5
├─ Gross margin per order: $3.25
├─ Payback period: $5 / $3.25 = 1.5 orders (good)
└─ LTV:CAC = $496 / $5 = 99:1 (excellent)
```

## Q8: How would you set up an experiment to optimize expansion?

### Answer:

**Experimentation Strategy:**

**Experiment 1: Onboarding Optimization**
```
Hypothesis: Better onboarding increases first order rate

Control: Current onboarding flow (20% conversion)
Variant A: Streamlined checkout (test)
Variant B: Incentive-based onboarding ($5 off first order)

Sample: 10K new users per group
Duration: 2 weeks
Metric: First order rate, first order AOV, D7 retention
Success: >25% conversion for one variant
```

**Experiment 2: Pricing**
```
Hypothesis: Lower prices increase adoption in price-sensitive market

Control: Standard pricing (30% commission)
Variant A: 25% commission (lower merchant cut)
Variant B: 35% commission (higher)

Sample: Different restaurants within each variant
Duration: 1 month
Metric: Order volume, merchant satisfaction, margins
Success: Find price elasticity, optimize unit economics
```

**Experiment 3: Feature Adoption**
```
Hypothesis: Loyalty program increases repeat orders

Control: No loyalty program
Variant A: Points-based loyalty (1 point per $1 spent)

Sample: 50% of users
Duration: 8 weeks
Metric: Repeat order rate, LTV, CAC payback
Success: >20% increase in repeat rate
```

## Summary: New Market/Vertical Expansion Strategy

1. **Validate market opportunity**: TAM, competition, regulatory, customer demand
2. **Design smart pilots**: Phase-gated approach, measure adoption & retention
3. **Track core metrics**: First-order conversion, D7/D30 retention, repeat rate, AOV, margins
4. **Monitor early warning signs**: Catch failures early, act fast
5. **Build unit economics model**: Understand CAC, LTV, margin path to profitability
6. **Experiment aggressively**: Optimize onboarding, pricing, features before scale
7. **Phase expansion**: Soft launch → beta → scale based on metrics hitting targets
