Storage and File formates

Table with 200 columns 1B rows

Annalyst reads 3 columns
Row based file format scans all rows and all column - Each row stored as chunk
Column based format scans only 3 columns - All data for 1 column is stored together


Parquet - Column Based - good for read heavy operations
Avro - row based - best for write heavy operations Eg for events ingestion , Each event is serailized in 1 single file


Why open table formats
When you dump raw parquet file in s3 - No acid guarantee, No schema enforcement, Small File Problem
